# AI編集部 オンボーディング

アトミックソフトウェア AI編集部へようこそ。
このガイドに沿って初回セットアップを済ませると、企画から原稿生成までをAIエージェントに任せられるようになります。

---

## この編集部でできること

企画書づくりから原稿生成、校正までを、入口の「編集部」エージェントと4羽の担当が分担します。

| エージェント | 役割 |
|---|---|
| editor（編集部・入口） | 「何をしたいか」を選択肢で尋ね、担当に振り分ける受付 |
| カワセミ（企画担当） | 企画テンプレから企画書を生成する |
| ハシボソカラス（ライティング担当） | 企画書と素材から原稿を生成する |
| オオタカ（校正担当） | 表記・構成・内容をチェックし修正する |
| シジュウカラ（最終レビュー担当） | 最終確認と、フィードバックの学習反映（PR提案）を行う |

利用者はエージェント名を覚える必要はありません。`@editor 編集部よろ` と入力すれば、あとは選択肢を選ぶだけです。

---

## 2つのロールと権限

| | 管理者（高根） | ユーザ（あなた） |
|---|---|---|
| GitHub | 編集・PR承認・マージ | 閲覧・PR作成 |
| Cowork / Claude Code | 利用可 | 利用可 |
| エージェント・スキルの改変 | できる | PR提案まで（マージは高根） |

エージェントの品質基準（スキルファイル）を実際に書き換えられるのは管理者だけです。
あなたのフィードバックはPR（修正提案）として管理者に届き、承認されると全員に反映されます。

## ① 初回セットアップ（一度だけ）

リポジトリは招待制（プライベート）です。セットアップには GitHub アカウントと認証が必要です。

不安な場合は、**セットアップ伴走プロンプト**（SlackやNotionで配布）をClaudeに貼れば、Claudeが1ステップずつ案内しながら代行してくれます。以下は手順の全体像です。

### STEP1: GitHub権限をリクエストする

1. GitHub アカウントを用意する（無ければ github.com で無料登録）
2. Slackの `#cp-pr-team-editorial` に、次の文面で権限を依頼する

```
🔑 GitHub権限依頼
@高根千聖 さん
cowork-agents リポジトリへの招待をお願いします。
GitHubユーザー名: 〇〇
（依頼者: あなたの名前）
```

3. 高根が招待すると、GitHubから招待メールが届くので「Accept invitation」で承認する

### STEP2: GitHub にログインする

GitHub CLI（gh）でログインします。

```bash
# gh が無ければインストール（Mac: brew install gh / Windows: winget install --id GitHub.cli）
gh auth login
```

「GitHub.com → HTTPS → ブラウザ認証」を選び、表示される8桁コードをブラウザのGitHub画面に入力します。

### STEP3: リポジトリを取り込み、セットアップする

```bash
gh repo clone Takanechis/cowork-agents ~/cowork-agents
cd ~/cowork-agents && bash setup.sh
```

setup.sh が自動で以下を行います。

- エージェントを `~/.claude/agents/` にリンク
- 起動時に `git pull` で自動同期するフックを設定

「✅ セットアップ完了」が出たら、Claude Code（または Cowork）を再起動してください。以降は起動のたびに最新のエージェントへ自動更新されます。

### STEP4: GitHub MCP を接続する（修正提案を出すため）

フィードバックを修正提案（PR）として送れるように、GitHubコネクタを接続します。

1. Cowork または Claude Code の設定からコネクタ一覧を開く
2. GitHub を選んで接続する
3. OAuth認証であなたのGitHubアカウントを承認する（スコープは `repo` を許可）

接続後、シジュウカラ（最終レビュー担当）があなたのアカウントでPRを作成できるようになります。PRの承認とマージは高根が行います。

### STEP5: 動作確認

Claude Code で次を入力し、編集部が選択肢を返せば成功です。

```
@editor 編集部よろ
```

## ② 運用マニュアル

使い方はひとつだけ覚えればOKです。

```
@editor 編集部よろ
```

すると編集部が「今日は何をしますか？」と選択肢を出します。番号で答えてください。

```
1. 企画を作る
2. 原稿を書く
3. 原稿を校正する
4. 完成した原稿をレビューして改善提案を出す
```

### フローA: 企画（選択肢 1）

1. 編集部が企画テンプレ（5項目）を返す
2. テンプレを埋めて送り返す
  - タイトル
  - テーマ
  - 読者（ターゲット）
  - 読者の読後感・便益
  - ポイントとなる要素
3. カワセミが企画書を生成する
4. 企画書を確認する。問題なければ次へ、直したい点があればフローCへ

### フローB: 原稿（選択肢 2・3）

1. 企画書と素材（文字起こし、資料、開催概要など）を渡す
2. ハシボソカラスが原稿を生成し、オオタカが校正する
3. 原稿を確認する。問題なければ完成、直したい点があればフローCへ

### フローC: フィードバックが学習になる仕組み（選択肢 4）

```
あなたが原稿への修正点を伝える
   ↓
シジュウカラがスキルファイルの改善案を生成する
   ↓
GitHub MCP で修正提案のPRを作成する（あなたのアカウントで）
   ↓
高根がPRをレビューする（承認者は高根のみ）
   ↓ マージ
全ユーザに git pull で自動反映される
```

あなたのフィードバックが、編集部全体の品質基準を育てます。

---

## 運用ルール

- 成果物（企画書・原稿）は共有ドライブに保存する
- 連絡や共有はSlackで行う
- エージェント・スキルは起動のたびに自動で最新版に更新される（GitHubと同期）

---

## 入れてはいけない情報

次のものはエージェントに渡さないでください。素材に含まれる場合は、その部分を消してから渡します。

- 患者・利用者の個人情報、個人の医療情報
- 社外秘の数字、未公開の経営情報
- ID・パスワードなどの認証情報

---

## 困ったとき

- エージェントが見えない → Claude Code を再起動する（起動フックで自動同期されます）
- 権限エラーが出る → STEP1のGitHub権限とSTEP4のGitHub MCP接続を確認する
- ログインでつまずく → セットアップ伴走プロンプトをClaudeに貼ると1ステップずつ案内されます
- 表記や品質の指摘がある → 遠慮なく伝えてください。シジュウカラがPRにして高根に届けます

ミッション「未来に、もっと余白を。」を、編集業務からも実現していきましょう。
