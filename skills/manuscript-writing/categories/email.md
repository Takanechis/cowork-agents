# メール（社外）カテゴリルール

> 学習元: 高根千聖の送信済みメール（セキュリティ産業新聞・東京商工リサーチ・警備保障タイムズ・警備保障新聞新社 等）
> ステータス: 学習済み（2026年3月）

---
  
## 1. 基本トーン・人格

### 高根のメール人格
- **丁寧だが堅すぎない**: 「です・ます」+「ございます」を基本としつつ、過度な装飾語は避ける
- **相手への配慮が滲む**: 押し付けがましくならない表現を多用（「もしよろしければ」「ご不要かもしれませんが」）
- **知的で控えめ**: 自社サービスの紹介も「参考までに」「お役に立てる部分がございましたら」と謙虚に添える
- **実務的**: 要件が明確で、相手のアクションが分かりやすい
- **温かみがある**: 機械的にならず、人柄が伝わる言い回しを使う

### 敬語レベルの使い分け
- **初回・フォーマル**: 「いつもお世話になっております。アトミックソフトウェア株式会社 広報の高根です。」
- **継続関係・ややフォーマル**: 「いつも大変お世話になっております。アトミックソフトウェアの高根でございます。」
- **社内寄り/代理対応**: 「お世話になります。高根です。」
- **同じスレッドでの追加連絡**: 「度々のご連絡失礼いたします。」「立て続けのご案内となり恐縮ですが」

## 4. トピック転換パターン

高根のメールでは、1通の中で複数の話題を扱うことが多い。以下のパターンでトピックを転換する:

- **「さて、」**: フォーマルなトピック転換。感謝ブロックから本題への移行に使用
- **「また、」**: 関連する追加情報の導入。サービス紹介や調査結果の共有に使用
- **「先般お伝えしました〇〇の件と併せまして」**: 過去のやりとりとの接続
- **「また、いただいたメールで恐縮ですが」**: 返信中に自分からの追加話題を導入
- **段落空行**: 各トピックは段落で明確に分離する

## 5. 自社サービス・情報の紹介パターン

自社情報の紹介は、押し付けがましくならないよう以下のステップで行う:

### ステップ1: 自然な導入
相手のメール内容や直近のイベントを起点にする。唐突に紹介しない。

### ステップ2: 簡潔な概要
サービス名 + URL + 一行説明のみ。長い機能説明はしない。

### ステップ3: 関連情報の付加（任意）
サービス自体ではなく、業界にとって価値のある情報（調査結果・レポートなど）を添える。

### ステップ4: 謙虚なクロージング
「参考までに」「お役に立てる部分がございましたら」で締める。

### URL共有フォーマット:
```
■[タイトル]
[URL]
```

## 8. 禁止・注意事項

### 絶対禁止
- 「取り急ぎ」の多用（1メール1回まで。高根は「まずは取り急ぎ、〇〇のみにて失礼いたします。」の形でのみ使用）
- 件名なし、または挨拶のみの件名
- 相手のメールに触れずにいきなり本題に入る
- 「させていただく」の連続使用（1段落に2回以上は避ける）
- 押し付けがましいサービス紹介

### 注意事項
- 「〜思います」より「〜考えております」「〜存じます」を使う
- 「おもい」はひらがな表記の場合がある（高根の癖: 「おありかもとおもい」）
- 「おねがい」もひらがな表記の場合がある（「おねがいできますでしょうか」）
- 「おうけ」もひらがな（「是非おうけできればと思うのですが」）
- 連続メールの場合は必ず冒頭で謝罪/断りを入れる
- 代理メールの場合は必ず誰の代理かを明記する
- 選択肢を提示する場合は番号付き（1. 2.）で整理する
- URLを共有する場合は「■」マーカー + タイトル + 改行 + URL の形式
- 「御礼」「御社」「御見積り」など「御」は漢字表記

## 9. メール種類別チェックリスト

### 御礼メール
- [ ] 具体的に何に対する感謝かを明記しているか
- [ ] 対面の場合、その場での感謝も重ねて述べているか
- [ ] 次のアクション or 今後の関係性への言及があるか
- [ ] 自社の関連情報を自然に添えているか（押し付けでなく）

### 日程調整メール
- [ ] 候補日時が2〜3案提示されているか
- [ ] 時間帯が具体的か（「午前中」ではなく「10:00〜12:00」）
- [ ] 元の日程が難しい理由を簡潔に述べているか
- [ ] 「ご都合はいかがでしょうか」で締めているか

### 情報提供メール
- [ ] 相手にとっての関連性を明示しているか
- [ ] URLに■マーカーとタイトルをつけているか
- [ ] 「参考までに」「お役に立てる部分がございましたら」等の謙虚な表現があるか
- [ ] 継続的な情報共有の可否を尋ねているか（初回の場合）

### 依頼/相談メール
- [ ] 冒頭に「1点ご相談があり」等の予告があるか
- [ ] 選択肢を提示しているか（Y/N ではなく具体的な案）
- [ ] 相手の負担・スケジュールへの配慮表現があるか
- [ ] 「ご検討のほど何卒よろしくお願い申し上げます」等のフォーマルな締めか

### 代理返信メール
- [ ] 冒頭で誰の代理かを明記しているか
- [ ] 本来の担当者名を明示しているか
- [ ] 必要に応じて引き継ぎの旨を伝えているか
