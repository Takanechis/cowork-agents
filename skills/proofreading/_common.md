# 校正スキル: 全種別共通の必須運用

## Overview

参照レギュレーション: skills/shared/regulation-notation.md
目的: 記事種別を問わず、すべての校正で必ず実施する共通運用を定義する。記事種別別の校正スキル（owned-media-interview.md 等）と併せて参照する。

このファイルは、部分校正で比喩表現や強調語を拾いきれず見逃しが繰り返し発生した事象を受けて整備した。校正のたびに本ファイルの2つの運用を必ず実施すること。

---

## 必須運用1: 比喩表現・強調語の必須チェックリスト

regulation-notation.md で禁止されている比喩表現・強調語・根拠なき修飾語は、校正のたびに以下のチェックリストで1項目ずつ原稿全文と照合する。該当があれば事実に即した表現へ書き換える修正案を必ずセットで示す。

### 比喩表現（事実や具体例に基づいた表現へ書き換える）

以下の語が比喩として使われていないか確認する。例示は代表例であり、これに限らない。

- 「壁」「壁を突破する」「突破する」（例: 壁を突破する鍵 → 解決の手がかり、課題を解決する方法）
- 「鍵」「カギ」（例: 成功の鍵 → 成功を左右する要因）
- 「複利で増える」「複利で賢くなる」（例: 知識が複利で増えていく → 知識が積み重なって増えていく）
- 「築き上げる」「土台を築く」（例: 信頼を築き上げる → 信頼を積み重ねる）
- 「配る」「全社へ配る」（例: ノウハウを全社へ配る → ノウハウを全社へ共有する）
- 「乗せる」「軌道に乗せる」（例: 事業を軌道に乗せる → 事業を安定させる、成長させる）
- 「最前線に立つ」「最前線」（例: 開発の最前線に立つ → 開発の中心を担う）
- 野球をはじめとするスポーツ由来の比喩（決定打、一塁を踏む、ホームランを狙う、三振、ストライク、土俵際 等。例: 決定打 → 決め手、最終的な要因）

### 強調語（削除するか、事実ベースの表現へ置き換える）

以下の語が根拠のない強調として使われていないか確認する。例示は代表例であり、これに限らない。

- 「非常に」（例: 非常に重要です → 重要です）
- 「最速で」（例: 最速で実装する → 短期間で実装する、◯日で実装する）
- 「どんどん」（例: どんどん改善する → 継続的に改善する）
- 「必ず」（例: 必ず成果が出る → 成果につながる）
- 「大きく」（例: 大きく変わる → 変わる、◯◯の点で変わる）
- 「一切」「一点集中」「完全に」（regulation-notation.md の禁止強調語）

### 根拠なき修飾語（削除する）

- 「画期的な」「革新的な」「唯一無二の」など、事実の裏づけがない修飾語は削除する。

## 必須運用2: 原稿全文を全項目で1パス点検する

部分校正で済ませず、最終的に必ず原稿全文をレギュレーション全項目で通し点検してから完了とする。

- 追記や一部修正のときも、修正箇所だけを見て完了としない。修正後の原稿全文を対象に、regulation-notation.md の全項目と記事種別スキルの全項目を、冒頭から末尾まで1パスで通し点検する。
- 必須運用1のチェックリスト（比喩表現・強調語・根拠なき修飾語）は、この全文1パス点検の中で必ず1項目ずつ照合する。
- 点検は「気になった箇所だけ拾う」進め方をしない。原稿の段落順に最初から最後まで目を通し、全項目を機械的に照合する。

## Output Rules

- 出力は Markdown で行う（** の使用は禁止）
- 指摘と修正案は1件ごとに必ずセットで出力する。指摘のみ・修正案のみの単独出力を禁止する
- 確認済み・問題なし・OKの項目は一切出力しない
- 言葉のトンマナは「ですます調」に統一する
