# 新機能リリース / 新サービス 構成テンプレート

ニュース種別: 新機能リリース / 新サービス
参照元: skills/press-release/SKILL.md

## セクション構成（この順序で生成すること）

1. タイトル
2. リード文
3. 背景・課題
4. 新機能 / サービスの概要
5. 導入効果
6. 代表者コメント（任意）
7. 提供開始時期・料金体系
8. 会社概要（formats/boilerplate.md を使用）
9. 問い合わせ先（formats/boilerplate.md を使用）

## 各セクションの記述ルール

### タイトル

- 30〜50字
- 新機能名・プロダクト名・効果を含む
- 動詞で締める（体言止め禁止）
- 「〜のご案内」「〜について」は禁止
- 数字または固有名詞を必ず含める

### リード文

- 3〜5文
- 5W1H を簡潔に網羅する（いつ・どこで・誰が・何を・なぜ・どのように）
- 「このたび」で始めてよい
- 新機能名・提供開始日・主な効果を含む

### 背景・課題（200〜300字）

- 業界の現状を数字・事実で示す
- この新機能 / サービスが解決する課題を明示する
- 根拠のない主観的な表現は禁止（事実と数字で語る）

### 新機能 / サービスの概要（300〜400字）

- 機能名・できること・特徴を3点で示す
- 具体的な操作や仕組みを平易な言葉で説明する
- プロダクト名は「medicalforce」（小文字始まり・一単語）で統一する

### 導入効果

- 数字で示す（例: 〇〇時間削減、〇〇%の工数削減）
- ビフォーアフターがある場合は明記する
- 数字の根拠がない場合は [要確認] と明示する

### 代表者コメント（任意・50〜100字）

- 短く、新機能 / サービスへの思いや今後の展開を端的に述べる
- 「代表取締役CEO／畠中 翔一」の形式で氏名・役職を明記する

### 提供開始時期・料金体系

- 提供開始日を明記する
- 料金: 無料 / 有料 / 要問い合わせ のいずれかを明示する
- 有料の場合は料金体系を記載する（または「詳細は問い合わせ」と明示）
