# プレスリリース共通スキル

このファイルは press-release-writer と press-release-reviewer の両エージェントが参照する共通ルールを定義する。
ニュース種別ごとの構成テンプレートは templates/ を、定型フォーマットは formats/boilerplate.md を参照すること。

## 対応ニュース種別とテンプレートファイル

| ニュース種別 | テンプレートファイル |
|---|---|
| 新機能リリース / 新サービス | templates/new-feature.md |
| 資金調達 | templates/funding.md |
| 提携・協業 | templates/partnership.md |
| 受賞・認定 | templates/award.md |

## 共通構成要素

すべてのニュース種別に共通して含める要素。

- タイトル: 30〜50字。数字・固有名詞・動詞を含む。「〜のご案内」「〜について」は禁止
- リード文: 3〜5文。5W1H を簡潔に網羅。「このたび」で始めてよい
- 背景・課題: 業界の現状と課題を数字・事実で示す（200〜300字）
- 詳細・ソリューション: 具体的な内容・数字・事実（300〜500字）
- コメント（任意）: 代表者コメント。50〜100字程度
- 会社概要: formats/boilerplate.md の定型フォーマットを使用
- 問い合わせ先: formats/boilerplate.md の定型フォーマットを使用

## 文体ルール

- 文体はです・ます調とし、断定的な表現を使う
- タイトルとリード文は体言止めを避け、動詞で締める
- 数字は算用数字を使い、単位を明記する（院、%、億円 等）
- 「させていただく」の過剰使用は禁止する（1記事2回まで）
- 根拠なき修飾語は禁止する（画期的な、革新的な、業界最高水準の 等）
- 誇大広告に該当しうる表現は禁止する（医療広告ガイドライン準拠）

## 禁止表現

- 「画期的な」「革新的な」「業界最高水準の」など根拠のない修飾語
- 「圧倒的な」「躍進的な」など自画自賛のニュアンスを持つ強調語
- 「させていただく」の連続使用
- AI特有の定型表現の多用（〜と言えるでしょう、〜ではないでしょうか 等）
- 「一切」「一点集中」「完全に」などの強調語
- 比喩的な表現
- 「——」を使った語句のつなぎ
- タイトルや見出しへの抽象的な表現
- 言葉同士を＝でつなぐ表現
- 並列の列挙に「・」を使う表現（「や」「と」「、」などで表現する）

## 表記ルール

agents/regulation-md-writing.md を必ず参照すること。以下はプレスリリース固有の重点項目。

- 会社名: 「アトミックソフトウェア株式会社」。初出時は「アトミックソフトウェア株式会社（旧：株式会社メディカルフォース）」と表記
- プロダクト名: 「medicalforce」（小文字始まり・一単語）、「警備フォース」
- 役員氏名: 姓と名の間に半角スペース（例: 畠中 翔一）
- 役職と氏名の区切り: 「／」を使用（例: 代表取締役CEO／畠中 翔一）
- 数字: 算用数字。全角・半角を統一する
- 単位を必ず明記する（院、%、億円 等）

## 品質ゲート（writer・reviewer 共通チェックリスト）

### 構成チェック

- タイトルが30〜50字で、数字または固有名詞を含み、動詞で締まっているか
- リード文が3〜5文で5W1H を網羅しているか
- セクション構成がニュース種別テンプレートに合った順序になっているか
- 全体の文量が600〜1200字程度か（会社概要・問い合わせ先を除く）
- 会社概要・問い合わせ先が formats/boilerplate.md の定型フォーマット通りか

### 表記チェック

- regulation-md-writing.md の全ルールに準拠しているか
- 会社名・プロダクト名・役員氏名の表記が正確か
- 禁止表現が含まれていないか
- 文体（です・ます調）が統一されているか
- 数字に単位が明記されているか
- 表記ゆれがないか（漢字・ひらがな・接続詞の統一）
- タイポ・誤字脱字がないか（regulation-writing-review.md のチェックリストも参照）

### 法的チェック

- 医療広告ガイドラインに抵触する表現がないか
- 「No.1」「業界初」等に調査出典が明記されているか
- 景品表示法に抵触する比較広告表現がないか

### PR TIMES 配信チェック

- Markdown記法（# ** - --- 等）がプレーンテキストに残っていないか
- 問い合わせ先の情報が完全か（担当者名・TEL・MAIL）

## 出力フォーマット

最終原稿はプレーンテキストで出力する。Markdown記法は一切使用しない。
改行と空行のみで構成を表現する。

フォーマット:

【タイトル】
（タイトル本文）

（リード文: 3〜5文）

■ （セクション見出し）
（本文）

■ （セクション見出し）
（本文）

（以降、ニュース種別テンプレートの構成に従いセクションを続ける）

（formats/boilerplate.md の会社概要・問い合わせ先を末尾に配置）
