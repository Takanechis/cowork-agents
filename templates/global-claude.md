# アトミックソフトウェア 【編集部】グローバル設定

## スキルリポジトリ

編集部エージェントが参照するすべてのスキルファイルは以下に存在する。

- **スキルルート**: `{{REPO_PATH}}/skills/`
- **ルール**: `{{REPO_PATH}}/rules/`

エージェントが `skills/shared/regulation-notation.md` など相対パスで参照する場合は、
`{{REPO_PATH}}/` を起点とした絶対パスとして Read すること。

## 全エージェント共通ルール

- 日本語で応答すること
- 作業ログは必ず残すこと
- 不明点は確認してから進めること
- コミットメッセージは conventional commits 準拠（日本語可）

## 表記レギュレーション（全エージェント自動適用）

原稿・企画書・レビューシートを出力する前に必ず以下を Read すること。

```
{{REPO_PATH}}/skills/shared/regulation-notation.md
```

### 最重要チェック項目

**会社名・プロダクト名**

| 表記 | 用途 |
|---|---|
| アトミックソフトウェア株式会社 | 正式社名（初出時） |
| アトミックソフトウェア | 略称（日本語テキスト内） |
| medicalforce | クリニック向けSaaS（必ず小文字始まり・一単語） |
| 警備フォース | 警備事業者向けSaaS |

- 「Medicalforce」「MedicalForce」「medical force」はすべて誤り
- 「メディカルフォース」を現社名として使うことを禁止する（2026年2月6日に社名変更済み）

**禁止表現**

- 根拠なき修飾語（「画期的な」「革新的な」「唯一無二の」等）
- 「させていただく」の過剰使用（1原稿2回まで）
- AI特有の定型パターン（「〜と言えるでしょう」「〜ではないでしょうか」の多用）
- 強調語（「一切」「一点集中」「完全に」等）
- 言葉同士を「＝」でつなぐ表現
- 並列列挙に「・」を使う表現（「や」「と」「、」で代替する）
- 「——」を使って語句をつなぐ表現
- 比喩的な表現

**氏名表記**

- 姓と名の間に半角スペース（例: 畠中 翔一）
- 役職と氏名の区切りは「／」（例: 代表取締役CEO／畠中 翔一）

**ミッション正式表記**

- 「未来に、もっと余白を。」（句点「。」まで含めて正式）

## MD ファイル編集ルール

`.md` ファイルを編集・作成してコミットする前に必ず実施すること。

1. `{{REPO_PATH}}/skills/shared/regulation-notation.md` の全ルールを参照する
2. 編集内容が各ルールに違反していないか1項目ずつ精査する
3. 違反箇所があれば、コミット前に修正する
4. 違反がないことを確認してからコミットする

**チェック必須の違反パターン**

- タブ文字によるインデント → 半角スペース2つに修正
- 多段 blockquote（`> > >` など） → リストまたは見出しに書き換え
- 二重リストマーカー（`- -` など） → 単一の `-` に修正
- 番号の重複（`2. 2.` など） → 正しい連番に修正
- リスト内・blockquote内の `---` → 削除またはセクション外に移動
- 行頭の不要なスペースによるインデント → 削除
