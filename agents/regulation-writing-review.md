## 5. ミッション・ビジョン・バリュー

### ミッション

「未来に、もっと余白を。」

※句点「。」まで含めて正式なミッション文。引用する際は鍵括弧「」で囲む。

### ビジョン（旧表現・スキルファイル記載）

「これからの産業の成長プロセスを合理化する」

### コアバリュー（4つ）

| バリュー名 | 読み・概要 |
|---|---|
| HRT | エイチアールティー。謙虚（Humility）・尊敬（Respect）・信頼（Trust）の頭文字。3要素をセットで説明するのが正式 |
| We | ウィー。自身の組織の役割に留まらず、一つ上のレイヤー（部門・事業全体）の戦略目標を深く理解し実現のために動く |
| 感動 | かんどう。思考と感情を重ね、単なる理解を超え相手に同化する。相手の期待を超える行動を自発的に行う |
| Punk | パンク。現状を疑い、自ら高い理想を描きつづける。自らの手で理想を具現化する実行者でありつづける |

バリューの表記ルール:
- 4つをまとめて列挙する場合: HRT / We / 感動 / Punk（スラッシュ区切り）
- Core Valueと呼ぶ場合もある（英語表記）

### 人事ポリシー（3つ）

| 表現 | 内容 |
|---|---|
| コアバリューファースト | 組織が変化し続けても、コアバリューを体現することを何より優先する |
| 10年で一生分の成長 | 短期間では実現できない壮大なビジョンに共感する仲間と長期的に成長し、実現を目指す |
| 透明性のある人組織 | チームとして高い成果を継続的にあげるために、強い信頼関係と透明性を土台とした相互理解を重視する |

## 6. 組織・制度の固有表現

| 表現 | 意味・補足 |
|---|---|
| スクラム | 全社的に導入している業務推進の仕組み。1スプリント＝2週間 |
| スプリント | 2週間単位のサイクル。「計画→実行→振り返り→再計画」を繰り返す |
| Daily Scrum | 毎日の進捗共有。問題の早期発見・適応のための場 |
| Sprint Review | スプリントの成果をレビューする場 |
| Retrospective | 前回のスプリントを振り返り、スクラム改善に取り組む場 |
| Refinement | やるべきこと自体を精査し、取り組むべき案件の解像度をあげる場 |
| Planning | 次のスプリントをどのように行うか計画する場 |
| グレード制度 | G1〜G9の9段階。キャリアパスはスペシャリストコース・マネジメントコースの2つ |
| スペシャリストコース | G6以上で分岐するキャリアパスの一方 |
| マネジメントコース | G6以上で分岐するキャリアパスの一方 |
| CxO | G9グレードの役職層を指す呼称 |
| 成果目標 | 評価制度の3目標のひとつ。事業計画から逆算し決める目標 |
| 組織貢献目標 | 評価制度の3目標のひとつ。組織におけるボトルネック解消のための目標 |
| 成長目標 | 評価制度の3目標のひとつ。Core Action（コアバリューを体現するための行動指針）を意識行動するための目標 |
| Core Action | コアバリューを体現するための行動指針。成長目標の評価軸 |
| フルフレックス制（コアタイムなし） | 勤務時間の制度。「フルフレックス」だけでなく「コアタイムなし」を必ず添える |
| ハイブリッド型 | 出勤・在宅を組み合わせた勤務形態。週2回はオフィスで顔合わせ |

## 13. 頻出フレーズ・言い回しパターン

以下は社内外の文書・インタビュー記事・メールで頻繁に使われる表現パターン。
原稿生成時はこれらの言い回しを積極的に参照すること。

### ミッション関連

- 「未来に、もっと余白を。」（ミッション文の完全形）
- 「余白をつくる」（ミッションの動詞表現）
- 「ソフトウェアの活用を通じて、さまざまな業務にかかる時間と手間を節約し、『余白』を生み出す」
- 「個人や企業が望む未来に向き合える環境をつくる」
- 「一人ひとりが自身の可能性と向き合える社会を実現する」

### 事業・戦略関連

- 「産業を1つにしぼらず」
- 「さまざまな産業のデータを集約し、業務を一元的に管理できる」
- 「各産業に最適化したシステムを構築」
- 「ニッチな産業であっても時間と手間を節約し業務効率化を実現」
- 「業務効率化 × 売上最大化を同時に実現」（medicalforce・警備フォース共通のキャッチフレーズ）
- 「産業のポテンシャルを解放する」
- 「いままで誰も解決してこなかった課題」
- 「誰も取り残さないDX」
- 「クリニック経営のための一元管理プラットフォーム」
- 「警備業における一元管理プラットフォーム」

### プロダクト説明

- 「予約・カルテ・会計・決済・CRM・在庫などが（バラバラに管理されている）」（課題の列挙パターン）
