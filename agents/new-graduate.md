## プロジェクト概要

アトミックソフトウェア株式会社の新卒採用専用LP（ランディングページ）を新規作成するプロジェクト。

## 与えたい印象

- **若手のうちから事業の中核を任される前提で新卒採用をしている会社**
  - 「新卒は全員事業部長候補」として見ているのが最大の特徴
  - 非連続的に事業立ち上げを予定しているため、ポジションが増え続ける＝チャンスが生まれ続ける
  - 早い段階から新規事業・事業責任者に挑戦できる環境がある
  - 新卒採用はまだ初期フェーズであり、一人ひとりを大切に育てていく

- **20代で圧倒的成長ができる環境**
  - 若いうちから複数ポジションや意思決定・事業推進に関われる
  - 若くして大きな裁量、若くして勝負できる環境がある

- **社会課題に正しく向き合っている会社**
  - テクノロジーが十分に届いていない産業の課題に向き合っている
  - 正解のない課題に挑み、自ら考え抜くことを楽しめる
  - 社会や産業をより良くする挑戦をしている

## デザイン方針

### 採用デザイン案
案3「グリッドデザインとオブジェクトの動き」をベースとする。

- バランスが良く、イラストの使い方が華やか
- 背景を白くするなど、色合いのバリエーションを追加して調整する
- AIエージェントのような先進的なイメージを取り入れ、技術力の高さを印象づける
- イラストは既存素材の活用・内製で対応（予算制約のため新規外注なし）

### UI / UX
- **モバイルファースト**で制作する（閲覧者の 8〜9 割がスマートフォン）
- 1 ページに情報を集約し、グローバルメニューからのページ内リンクで各セクションへ遷移
- スマホでの離脱防止を最優先とする

## web-builder エージェントとの連携

このプロジェクトは `web-builder` エージェントが管轄する **corporate.v4** リポジトリ上で実装する。
実装時は web-builder の規約（ブランチ運用・Tailwind CSS コーディング規約・ページテンプレートパターン等）に必ず従うこと。

### recruit 系ページの実装ルール（web-builder より）
- テンプレート: `header-recruit.php` / `footer-recruit.php` / `navigation-recruit.php` を使用
- セクションコンポーネント配置先: `components/section/recruit/`
- ページ固有 JS: `assets/js/pages/recruit-new-graduate.js`
- ブランチ: `feature/page-recruit-new-graduate` から作業開始

### 実装フロー
1. `test` ブランチから `feature/page-recruit-new-graduate` を作成
2. Figma デザイン（案 3 ベース）を確認し、セクション構成を決定
3. `page-recruit-new-graduate.php` テンプレートを作成
4. `components/section/recruit/` 配下に各セクションコンポーネントを追加
5. モバイルファーストで Tailwind CSS を実装
6. `npm run build` でビルド → Docker 環境で動作確認（localhost:8000）
7. `test` ブランチへ PR 作成
